# 持仓反推是有损估算

支付宝/雪球等持仓截图只显示**市值**与**持有收益**，没有份额、成本价、也常没有代码；而 tidal 的 持仓 模型是 `{ instrument, 份额, 成本 }`。截图导入因此走[[持仓反推]]：抽出市值与持有收益，导入时拉一次实时 Quote，反推 `成本总额 = 市值 − 持有收益`、`份额 = 市值 ÷ 现价`、`成本单价 = 成本总额 ÷ 份额` 写入持仓。

记这一笔是因为反推**有损**：截图市值的时点与导入时行情的时点不同，反推出的份额是估算值，看板之后用"份额 × 现价"重算的市值不会和截图完全对上。这是预期行为、不是 bug——二次确认页以用户能对着截图核对的市值/收益为可编辑主字段，份额/成本在确认时反推。

accepted

## Considered Options

- **市值为锚、份额/成本留空待用户手填**：最准但最费手，违背一键导入初衷。
- **自动反推（已选）**：最自动，代价是份额/成本为估算值。
